My Reservations — recurring card layouts

Same data shown four ways. Each layout renders the master card plus a recurring series with 1 modified, 1 cancelled, and 1 added occurrence (top), and a stress test with 10 deviations (bottom) so you can see how each scales. Hover the rows to see clickable affordances.

1 — Tree of child sub-cards 2 — Always-visible inline section 3 — Tabbed footer 4 — Default-open collapsible panel

1 Tree of child sub-cards (refined)

A typographic anchor inside the master card (“Occurrences that differ from this series · 3”) serves as the trunk's origin. The line is now a single continuous mark that crosses the master's bottom border. Branches use rounded L-corners instead of right-angle stubs. Children are quieted: hairline border, soft tinted background, no shadow — reading as annotations of the master rather than independent cards.

What changedTrunk anchored to a typographic label inside the master (no float). Single continuous spine across the card boundary. Rounded L-corners. Children visually subordinate (no shadow, soft tint). Cancelled rows now use a dashed border to communicate their non-interactive state.
Watch forVertical space still grows linearly with the number of overrides — the stress test below shows how a 10-override series looks.

2 Always-visible inline section

Permanent “Modified Occurrences” section inside the master card, separated by a divider. Always visible — zero discovery cost. For series with many deviations, only the first 3 render with a “Show all N” button.

ProsMost prominent. No clicks required. Stays inside the master card so the connection is implicit.
ConsMaster card gets taller. “Show all” mitigates but adds another control.

3 Tabbed footer

Tabs at the bottom of the master card pivot between “Details” (extra info) and “Occurrences (N)”. Card height never changes regardless of how many deviations exist.

ProsCard height constant. Count visible in the tab. Works equally well for 1 or 100 deviations.
ConsAdds chrome to a card that didn't have any. One click required to discover overrides.

Scenario B — 10 deviations (Occurrences tab active)

4 Default-open collapsible panel

Like option 2 but with a header that toggles. Opens by default for masters with overrides, closes if the user collapses it. The summary line stays visible when collapsed so the count is never hidden.

ProsDiscoverable by default. User can collapse if they want a tighter list. Compromise between always-visible and dropdown.
ConsOne more interactive element. Slightly less obvious than option 1's tree if the user has never seen the chevron pattern.
