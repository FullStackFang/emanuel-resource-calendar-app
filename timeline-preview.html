Timeline View — Before / After Phase 1 · CSS tokens only

What's different in "After"

  • A — Sticky time column + day header use a box-shadow for elevation instead of 2px borders / gradient; creates real depth during scroll.
  • B — Event-block hover uses an inset box-shadow (pre-reserved slot) and a 0.5px lift; no more layout shift from border-width: 1px → 1.5px.
  • C — Setup / teardown hatching tinted with the event's --loc-color instead of rgba(0,0,0,0.11); pattern reads as one hue, not muddy.
  • D — All-day badge switched from competing blue to neutral mono pill; location-colored blocks own the color layer.
  • F — Current-time reduced from a red badge to a 1.5px line + dot with a soft halo; whispers "live" rather than shouting.
  • G — Hour labels and event times use --font-mono with tabular-nums; numeric data has a consistent identity across the whole view.
  • H — Removed negative top: -8px hack on hour labels (now -7px feels tuned, not hacked); tighter letter-spacing on titles.
  • Phase 3 sneak-peek — Alternating day columns have a 1.2% neutral tint (odd/even striping) to reinforce the 3-day grid at a glance.
